## 1. The AI Testing Landscape

AI is transforming test automation in several ways — from generating boilerplate to autonomously navigating UIs. Understanding the layers helps you pick the right tool for the right job.

### The Four Layers of AI in Testing

```
┌─────────────────────────────────────────────────────────┐
│  Layer 4: Autonomous Agents                             │
│  AI decides WHAT to test, navigates UI, writes & runs   │
│  tests entirely on its own                              │
├─────────────────────────────────────────────────────────┤
│  Layer 3: Self-healing                                  │
│  AI detects broken locators and fixes them              │
│  automatically at runtime or in CI                      │
├─────────────────────────────────────────────────────────┤
│  Layer 2: AI-assisted Authoring                         │
│  AI generates test code from prompts, specs, or         │
│  recordings — human reviews and commits                 │
├─────────────────────────────────────────────────────────┤
│  Layer 1: AI Tooling / Copilot                          │
│  GitHub Copilot, Claude, ChatGPT in your IDE            │
│  autocomplete, explain, refactor test code              │
└─────────────────────────────────────────────────────────┘
```

### Where Each Approach Fits

| Approach | Speed | Reliability | Human Effort | Best For |
|---|---|---|---|---|
| Copilot / Claude in IDE | Fast | High | Low | Daily authoring |
| `playwright codegen` | Fast | Medium | Medium | Bootstrapping |
| AI test generation | Very fast | Medium | Review needed | Large suites |
| Self-healing locators | Automatic | High | Very low | Maintenance |
| Autonomous agents | Slow | Variable | Oversight needed | Exploration |

## 2. AI-powered Code Generation

### `playwright codegen` — Built-in Recorder

Playwright ships with a built-in code recorder. It watches your actions and generates typed Playwright code.

```bash
# Open codegen for a URL
npx playwright codegen https://example.com

# With specific browser
npx playwright codegen --browser=firefox https://example.com

# Save output to file
npx playwright codegen --output=tests/recorded.spec.ts https://example.com

# With viewport
npx playwright codegen --viewport-size=1280,720 https://example.com

# With saved auth state
npx playwright codegen --load-storage=playwright/.auth/user.json https://app.com

# Record in a specific language
npx playwright codegen --target=python https://example.com
npx playwright codegen --target=java https://example.com
```

The recorder generates code like this — use it as a starting point, not final output:

```typescript
// Auto-generated by playwright codegen — REFACTOR before committing
import { test, expect } from '@playwright/test';

test('test', async ({ page }) => {
  await page.goto('https://example.com/login');
  await page.locator('#email').click();           // ← fragile: use getByLabel
  await page.locator('#email').fill('user@test.com');
  await page.locator('#password').fill('pass123');
  await page.locator('button').nth(2).click();    // ← fragile: use getByRole
  await expect(page).toHaveURL('https://example.com/dashboard');
});
```

Refactored version:

```typescript
// ✅ After AI-assisted refactoring
import { test, expect } from '@playwright/test';

test('user can log in with valid credentials', async ({ page }) => {
  await page.goto('/login');
  await page.getByLabel('Email').fill('user@test.com');
  await page.getByLabel('Password').fill('pass123');
  await page.getByRole('button', { name: 'Sign in' }).click();
  await expect(page).toHaveURL('/dashboard');
});
```

### Prompting Claude/LLM to Generate Tests

The most productive AI workflow for test generation: give the AI structured context and ask for structured output.

#### Prompt Pattern 1: From User Story

```
You are a Playwright test engineer using TypeScript.

Given the following user story:
---
As a registered user, I want to reset my password so that I can
regain access if I forget it.

Acceptance criteria:
- User can navigate to "Forgot password" from the login page
- User enters their email and receives a confirmation message
- Invalid email format shows an inline validation error
- Non-existent email shows "If this email exists, we'll send a link"
---

Write a Playwright test file using:
- @playwright/test with TypeScript
- getByRole / getByLabel / getByTestId locators
- test.describe grouping
- beforeEach for navigation
- One test per acceptance criterion
- Meaningful test names
```

#### Prompt Pattern 2: From Existing Page Object

```
Given this Page Object:

```typescript

  readonly addressInput: Locator;
  readonly cityInput: Locator;

    this.addressInput = page.getByLabel('Street address');
    this.cityInput = page.getByLabel('City');
    this.placeOrderButton = page.getByRole('button', { name: 'Place order' });
    this.orderConfirmation = page.getByTestId('order-confirmation');

Write tests covering:
1. Happy path — complete checkout
2. Missing required fields — validation errors
3. Invalid email format
4. Successful order shows confirmation number

## 3. Smart Locator Strategies

Smart locators use multiple fallback strategies to find elements, making tests resilient to minor HTML changes.

### The Multi-Strategy Locator Pattern

```typescript
// utils/smartLocator.ts
import { Page, Locator } from '@playwright/test';

interface LocatorStrategy {
  strategy: string;
  selector: string | { role?: string; name?: string; testId?: string };
}

/**
 * Tries multiple locator strategies in priority order.
 * Falls back gracefully if the primary locator fails.
 */
export async function smartLocate(
  page: Page,
  strategies: LocatorStrategy[],
  timeout = 5000
): Promise<Locator> {
  for (const { strategy, selector } of strategies) {
    try {
      let locator: Locator;

      if (strategy === 'testId' && typeof selector === 'string') {
        locator = page.getByTestId(selector);
      } else if (strategy === 'role' && typeof selector === 'object') {
        locator = page.getByRole(selector.role as any, { name: selector.name });
      } else if (strategy === 'label' && typeof selector === 'string') {
        locator = page.getByLabel(selector);
      } else if (strategy === 'css' && typeof selector === 'string') {
        locator = page.locator(selector);
      } else {
        continue;
      }

      await locator.waitFor({ state: 'attached', timeout });
      console.log(`✅ Locator found using strategy: ${strategy}`);
      return locator;

    } catch {
      console.warn(`⚠️ Strategy "${strategy}" failed, trying next...`);
    }
  }

  throw new Error(`All locator strategies exhausted for: ${JSON.stringify(strategies)}`);
}

// Usage
const submitButton = await smartLocate(page, [
  { strategy: 'testId',  selector: 'submit-btn' },
  { strategy: 'role',    selector: { role: 'button', name: 'Submit' } },
  { strategy: 'css',     selector: 'button[type="submit"]' },
]);

await submitButton.click();
```

### AI-powered Locator Repair at Runtime

```typescript
// utils/aiLocatorRepair.ts
import Anthropic from '@anthropic-ai/sdk';
import { Page, Locator } from '@playwright/test';

const client = new Anthropic();

/**
 * When a locator fails, capture the page HTML and ask Claude
 * to suggest a new working selector.
 */
export async function repairLocator(
  page: Page,
  brokenSelector: string,
  elementDescription: string
): Promise<Locator> {
  console.warn(`🔧 Attempting AI repair for broken selector: ${brokenSelector}`);

  // Capture current page state
  const pageHTML = await page.content();
  const truncatedHTML = pageHTML.slice(0, 8000); // token limit

  const response = await client.messages.create({
    model: 'claude-opus-4-6',
    max_tokens: 512,
    messages: [
      {
        role: 'user',
        content: `You are a Playwright locator expert.

The selector "${brokenSelector}" no longer works on this page.
I'm looking for: "${elementDescription}"

Here is the current page HTML:
\`\`\`html
${truncatedHTML}
\`\`\`

Return ONLY a JSON object with this shape:
{
  "strategy": "testId" | "role" | "label" | "text" | "css",
  "selector": "the selector string",
  "confidence": 0.0 to 1.0,
  "reason": "brief explanation"
}`,
      },
    ],
  });

  const raw = response.content[0].type === 'text' ? response.content[0].text : '{}';
  const result = JSON.parse(raw) as {
    strategy: string;
    selector: string;
    confidence: number;
    reason: string;
  };

  console.log(`🤖 AI suggestion: [${result.strategy}] "${result.selector}" (confidence: ${result.confidence})`);
  console.log(`   Reason: ${result.reason}`);

  // Build and return the repaired locator
  switch (result.strategy) {
    case 'testId': return page.getByTestId(result.selector);
    case 'label':  return page.getByLabel(result.selector);
    case 'text':   return page.getByText(result.selector);
    case 'role': {
      const [role, name] = result.selector.split('|');
      return page.getByRole(role as any, { name });
    }
    default: return page.locator(result.selector);
  }
}
```

## 4. Self-healing Tests

Self-healing tests automatically detect and recover from broken locators — either at runtime or as a CI post-processing step.

### Runtime Self-healing Wrapper

```typescript
// utils/selfHealingPage.ts
import { Page, Locator } from '@playwright/test';
import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic();

interface HealingRecord {
  original: string;
  healed: string;
  timestamp: Date;
  confidence: number;
}

export class SelfHealingPage {
  private readonly page: Page;
  private readonly healingLog: HealingRecord[] = [];

  constructor(page: Page) {
    this.page = page;
  }

  /**
   * Attempts to locate an element, falling back to AI repair if it fails.
   */
  async locate(
    selector: string,
    description: string,
    timeout = 5000
  ): Promise<Locator> {
    const locator = this.page.locator(selector);

    try {
      await locator.waitFor({ state: 'attached', timeout });
      return locator;
    } catch {
      console.warn(`⚠️ Selector broken: "${selector}". Attempting self-heal...`);

      const healed = await this.healLocator(selector, description);

      this.healingLog.push({
        original: selector,
        healed,
        timestamp: new Date(),
        confidence: 0.8,
      });

      return this.page.locator(healed);
    }
  }

  private async healLocator(broken: string, description: string): Promise<string> {
    const html = (await this.page.content()).slice(0, 6000);

    const res = await client.messages.create({
      model: 'claude-opus-4-6',
      max_tokens: 256,
      messages: [{
        role: 'user',
        content: `Selector "${broken}" is broken. Find "${description}" in this HTML.
Return ONLY the new CSS selector string, nothing else.

HTML:
${html}`,
      }],
    });

    return res.content[0].type === 'text' ? res.content[0].text.trim() : broken;
  }

  /**
   * Outputs all healing actions taken during the session.
   * Use in afterAll to update your test files.
   */
  getHealingReport(): HealingRecord[] {
    return this.healingLog;
  }

  printHealingReport(): void {
    if (this.healingLog.length === 0) {
      console.log('✅ No locator healing was required.');
      return;
    }

    console.log('\n🔧 Self-healing Report:');
    console.log('─'.repeat(60));
    for (const record of this.healingLog) {
      console.log(`Original : ${record.original}`);
      console.log(`Healed   : ${record.healed}`);
      console.log(`Time     : ${record.timestamp.toISOString()}`);
      console.log('─'.repeat(60));
    }
  }
}
```

### Offline Self-healing: Post-CI Selector Update

A different approach — detect broken selectors in CI and open a PR with fixes.

```typescript
// scripts/healSelectors.ts
import * as fs from 'fs';
import * as path from 'path';
import Anthropic from '@anthropic-ai/sdk';

interface BrokenSelector {
  file: string;
  line: number;
  selector: string;
  error: string;
}

const client = new Anthropic();

/**
 * Parses Playwright's JSON report for locator timeout errors,
 * then asks Claude to suggest fixes.
 */
async function healFromReport(reportPath: string): Promise<void> {
  const report = JSON.parse(fs.readFileSync(reportPath, 'utf-8'));
  const broken: BrokenSelector[] = extractBrokenSelectors(report);

  if (broken.length === 0) {
    console.log('✅ No broken selectors found in report.');
    return;
  }

  console.log(`🔧 Found ${broken.length} broken selectors. Requesting AI fixes...`);

  for (const item of broken) {
    const sourceCode = fs.readFileSync(item.file, 'utf-8');

    const response = await client.messages.create({
      model: 'claude-opus-4-6',
      max_tokens: 1024,
      messages: [{
        role: 'user',
        content: `This Playwright locator is failing:
File: ${item.file} (line ${item.line})
Selector: ${item.selector}
Error: ${item.error}

Source file context:
\`\`\`typescript
${sourceCode}
\`\`\`

Suggest a more resilient replacement locator.
Return ONLY the replacement selector string.`,
      }],
    });

    const suggestion = response.content[0].type === 'text'
      ? response.content[0].text.trim()
      : item.selector;

    console.log(`\nFile: ${item.file}:${item.line}`);
    console.log(`  Old: ${item.selector}`);
    console.log(`  New: ${suggestion}`);
  }
}

function extractBrokenSelectors(report: any): BrokenSelector[] {
  // Parse playwright JSON report structure
  const broken: BrokenSelector[] = [];
  for (const suite of report.suites ?? []) {
    for (const spec of suite.specs ?? []) {
      for (const test of spec.tests ?? []) {
        for (const result of test.results ?? []) {
          if (result.status === 'failed' && result.error?.message?.includes('locator')) {
            broken.push({
              file: spec.file,
              line: spec.line,
              selector: extractSelectorFromError(result.error.message),
              error: result.error.message,
            });
          }
        }
      }
    }
  }
  return broken;
}

function extractSelectorFromError(message: string): string {
  const match = message.match(/locator\('(.+?)'\)/);
  return match ? match[1] : 'unknown';
}
```

## 5. Claude / LLM as a Test Agent

An AI agent can autonomously explore a web application, decide what to test, and execute tests — all driven by natural language instructions.

### Simple Agent Loop

```typescript
// agent/testAgent.ts
import { chromium, Page } from 'playwright';
import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic();

interface AgentAction {
  type: 'click' | 'fill' | 'navigate' | 'assert' | 'done';
  selector?: string;
  value?: string;
  url?: string;
  assertion?: string;
  reasoning: string;
}

/**
 * A minimal AI agent that explores a URL and performs actions
 * based on Claude's decisions.
 */
async function runTestAgent(
  startUrl: string,
  goal: string,
  maxSteps = 10
): Promise<void> {
  const browser = await chromium.launch({ headless: false });
  const page = await browser.newPage();
  await page.goto(startUrl);

  const conversationHistory: Anthropic.MessageParam[] = [];
  let step = 0;

  while (step < maxSteps) {
    step++;

    // Capture current page state
    const screenshot = await page.screenshot({ encoding: 'base64' });
    const url = page.url();
    const title = await page.title();

    console.log(`\n🤖 Step ${step} — URL: ${url}`);

    // Ask Claude what to do next
    const userMessage: Anthropic.MessageParam = {
      role: 'user',
      content: [
        {
          type: 'image',
          source: { type: 'base64', media_type: 'image/png', data: screenshot },
        },
        {
          type: 'text',
          text: `Current URL: ${url}
Page title: ${title}
Step: ${step}/${maxSteps}

Goal: ${goal}

Based on the screenshot, what is the SINGLE next best action to achieve the goal?
Return ONLY a JSON object:
{
  "type": "click" | "fill" | "navigate" | "assert" | "done",
  "selector": "CSS selector or accessible name",
  "value": "text to fill (if type=fill)",
  "url": "URL to navigate to (if type=navigate)",
  "assertion": "what to verify (if type=assert)",
  "reasoning": "why this action"
}`,
        },
      ],
    };

    conversationHistory.push(userMessage);

    const response = await client.messages.create({
      model: 'claude-opus-4-6',
      max_tokens: 512,
      system: `You are an expert QA engineer controlling a browser via Playwright.
You receive screenshots and decide what actions to take to achieve a testing goal.
Always choose the most reliable, accessible selector (getByRole, getByLabel preferred).
Be methodical — one step at a time.`,
      messages: conversationHistory,
    });

    const rawAction = response.content[0].type === 'text' ? response.content[0].text : '{}';

    let action: AgentAction;
    try {
      action = JSON.parse(rawAction.replace(/```json|```/g, '').trim());
    } catch {
      console.error('Failed to parse agent response:', rawAction);
      break;
    }

    console.log(`   Action: ${action.type} — ${action.reasoning}`);

    // Add assistant response to history
    conversationHistory.push({ role: 'assistant', content: rawAction });

    // Execute the action
    const result = await executeAction(page, action);

    if (action.type === 'done') {
      console.log('\n✅ Agent completed the goal!');
      break;
    }

    if (!result.success) {
      console.warn(`⚠️ Action failed: ${result.error}`);
    }
  }

  await browser.close();
}

async function executeAction(
  page: Page,
  action: AgentAction
): Promise<{ success: boolean; error?: string }> {
  try {
    switch (action.type) {
      case 'navigate':
        await page.goto(action.url!);
        break;

      case 'click':
        // Try accessible name first, fall back to CSS
        try {
          await page.getByRole('button', { name: action.selector! }).click({ timeout: 3000 });
        } catch {
          await page.locator(action.selector!).click({ timeout: 5000 });
        }
        break;

      case 'fill':
        try {
          await page.getByLabel(action.selector!).fill(action.value ?? '');
        } catch {
          await page.locator(action.selector!).fill(action.value ?? '');
        }
        break;

      case 'assert':
        const text = await page.locator('body').textContent();
        if (!text?.includes(action.assertion ?? '')) {
          return { success: false, error: `Assertion failed: "${action.assertion}" not found` };
        }
        break;
    }

    await page.waitForLoadState('networkidle', { timeout: 5000 }).catch(() => {});
    return { success: true };

  } catch (error) {
    return { success: false, error: String(error) };
  }
}

// Run the agent
runTestAgent(
  'https://demo.playwright.dev/todomvc',
  'Add three todo items, mark one as complete, then verify the active count shows 2',
);
```

async function generateTestFile(spec: TestSpec): Promise<void> {

Generate production-quality TypeScript test files.

- @playwright/test imports
- getByRole, getByLabel, getByTestId for locators

## 7. Computer Use Agent for Testing

Anthropic's Computer Use feature lets Claude directly control a computer — including a browser — without needing pre-written selectors. This is the most autonomous form of AI testing.

### Architecture

```
┌──────────────────────────────────────────────────────────────┐
│                     Your Orchestrator                        │
│                                                              │
│  1. Take screenshot  ──→  Claude API (claude-opus-4-6)       │
│                     ←──  Tool call: { type: "click", x, y } │
│  2. Execute action                                           │
│  3. Take screenshot  ──→  Claude API                         │
│                     ←──  Tool call: { type: "type", text }  │
│  4. Repeat until done or max steps reached                   │
└──────────────────────────────────────────────────────────────┘
```

### Minimal Computer Use + Playwright

```typescript
// agent/computerUseAgent.ts
import Anthropic from '@anthropic-ai/sdk';
import { chromium, Page } from 'playwright';

const client = new Anthropic();

const TOOLS: Anthropic.Tool[] = [
  {
    name: 'computer',
    type: 'computer_20241022' as any,
    input_schema: {
      type: 'object',
      properties: {},
      required: [],
    },
    // @ts-expect-error — beta tool fields
    display_width_px: 1280,
    display_height_px: 720,
  } as any,
];

async function runComputerUseAgent(
  page: Page,
  task: string,
  maxIterations = 15
): Promise<void> {
  const messages: Anthropic.MessageParam[] = [
    {
      role: 'user',
      content: task,
    },
  ];

  for (let i = 0; i < maxIterations; i++) {
    const screenshot = await page.screenshot({ encoding: 'base64' });

    // Inject current screenshot as a tool result context
    if (i > 0) {
      messages.push({
        role: 'user',
        content: [
          {
            type: 'tool_result',
            tool_use_id: `step_${i}`,
            content: [
              {
                type: 'image',
                source: { type: 'base64', media_type: 'image/png', data: screenshot },
              },
            ],
          } as any,
        ],
      });
    }

    const response = await client.messages.create({
      model: 'claude-opus-4-6',
      max_tokens: 1024,
      tools: TOOLS,
      messages,
      system: 'You are controlling a browser to complete a QA testing task. Be precise and methodical.',
      betas: ['computer-use-2024-10-22'] as any,
    });

    messages.push({ role: 'assistant', content: response.content });

    if (response.stop_reason === 'end_turn') {
      console.log('✅ Agent completed task.');
      break;
    }

    // Process tool calls
    for (const block of response.content) {
      if (block.type !== 'tool_use') continue;
      if (block.name !== 'computer') continue;

      const input = block.input as {
        action: string;
        coordinate?: [number, number];
        text?: string;
        key?: string;
      };

      await executeComputerAction(page, input);
    }
  }
}

async function executeComputerAction(
  page: Page,
  action: { action: string; coordinate?: [number, number]; text?: string; key?: string }
): Promise<void> {
  console.log(`🖥️  Action: ${action.action}`, action.coordinate ?? action.text ?? '');

  switch (action.action) {
    case 'screenshot':
      break; // handled above

    case 'left_click':
    case 'right_click':
    case 'middle_click': {
      const [x, y] = action.coordinate!;
      const button = action.action === 'right_click' ? 'right'
                   : action.action === 'middle_click' ? 'middle' : 'left';
      await page.mouse.click(x, y, { button });
      break;
    }

    case 'double_click': {
      const [x, y] = action.coordinate!;
      await page.mouse.dblclick(x, y);
      break;
    }

    case 'type':
      await page.keyboard.type(action.text ?? '');
      break;

    case 'key':
      await page.keyboard.press(action.key ?? '');
      break;

    case 'mouse_move': {
      const [x, y] = action.coordinate!;
      await page.mouse.move(x, y);
      break;
    }

    case 'scroll': {
      const [x, y] = action.coordinate!;
      await page.mouse.wheel(0, y > 360 ? 300 : -300);
      break;
    }
  }

  await page.waitForLoadState('networkidle', { timeout: 3000 }).catch(() => {});
}

// Example usage
(async () => {
  const browser = await chromium.launch({ headless: false });
  const page = await browser.newPage();
  await page.setViewportSize({ width: 1280, height: 720 });
  await page.goto('https://demo.playwright.dev/todomvc');

  await runComputerUseAgent(
    page,
    'Add "Buy milk", "Walk the dog", and "Call dentist" as todo items. Mark "Buy milk" as done. Verify 2 items remain active.',
  );

  await browser.close();
})();
```

## 8. AI-assisted Test Maintenance

### Batch Locator Migration

```typescript
// tools/migrateLocators.ts
import Anthropic from '@anthropic-ai/sdk';
import * as fs from 'fs';
import * as glob from 'glob';

const client = new Anthropic();

/**
 * Finds all spec files and asks Claude to modernize their locators.
 * Old: page.$('#id'), page.locator('.class').nth(0)
 * New: page.getByRole(), page.getByLabel(), page.getByTestId()
 */
async function migrateSpecFiles(pattern = 'tests/**/*.spec.ts'): Promise<void> {
  const files = glob.sync(pattern);
  console.log(`Found ${files.length} spec files to migrate.`);

  for (const file of files) {
    const original = fs.readFileSync(file, 'utf-8');

    // Only process files with old-style locators
    if (!original.includes('page.$') && !original.includes('.nth(0)')) {
      console.log(`⏭️  Skipping ${file} (already modern)`);
      continue;
    }

    console.log(`🔄 Migrating ${file}...`);

    const response = await client.messages.create({
      model: 'claude-opus-4-6',
      max_tokens: 4096,
      messages: [{
        role: 'user',
        content: `Migrate this Playwright spec file to use modern locators.

Rules:
- Replace page.$() → page.locator() or semantic locators
- Replace .nth(0) → .first() where appropriate
- Replace CSS class selectors with getByRole/getByLabel/getByTestId when possible
- Keep the logic identical, only update locators
- Preserve all comments and test names

\`\`\`typescript
${original}
\`\`\`

Return ONLY the migrated TypeScript code, no explanation.`,
      }],
    });

    const migrated = response.content[0].type === 'text'
      ? response.content[0].text.replace(/```typescript\n?|```\n?/g, '').trim()
      : original;

    // Write backup
    fs.writeFileSync(`${file}.bak`, original);
    fs.writeFileSync(file, migrated);

    console.log(`✅ Migrated: ${file}`);
  }
}

migrateSpecFiles();
```

## 9. Prompt Engineering for Test Automation

Getting the best output from AI requires precise, structured prompts. These patterns work consistently.

### The STAR Prompt Template

**S**ituation → **T**ask → **A**ction → **R**esult

```
Situation: You are a Playwright test engineer working on an e-commerce checkout flow.

Task: Write a test that verifies a user can apply a discount coupon during checkout.

Action: The test should:
1. Navigate to the cart page
2. Enter coupon code "SAVE20" in the coupon field
3. Click "Apply coupon"
4. Assert the discount line shows "-20%"
5. Assert the total is reduced accordingly

Result: Return a TypeScript test using @playwright/test,
getByRole/getByLabel locators, and meaningful assertion messages.
```

### Role + Constraint Prompting

```
You are a senior QA automation engineer with 10 years of Playwright experience.
You write minimal, readable, maintainable test code.

HARD CONSTRAINTS:
- Never use page.waitForTimeout()
- Never use CSS class selectors unless there is no alternative
- Always add { timeout: X } overrides to slow assertions
- Always use TypeScript strict types — no `any`
- Every assertion must have a custom failure message

Given: [your request here]
```

### Chain-of-thought for Complex Scenarios

```
Think step by step before writing the test.

1. First, identify all the user-visible elements on the page that need interaction
2. For each element, choose the most resilient locator strategy
3. Identify the assertions needed to verify correct behavior
4. Consider edge cases: empty states, loading states, error states
5. THEN write the final test code

Feature to test: [description]
```

### Few-shot Prompting

Providing examples dramatically improves output quality:

```
Write a Playwright test following the same style as these examples.

Example 1 (good):
\`\`\`typescript
test('should display error when email is missing', async ({ page }) => {
  await page.goto('/register');
  await page.getByRole('button', { name: 'Create account' }).click();
  await expect(
    page.getByRole('alert').filter({ hasText: 'Email is required' }),
    'Email validation error should appear'
  ).toBeVisible();
});
\`\`\`

Example 2 (good):
\`\`\`typescript
test('should filter products by category', async ({ page }) => {
  await page.goto('/shop');
  await page.getByRole('combobox', { name: 'Category' }).selectOption('Electronics');
  const cards = page.getByTestId('product-card');
  await expect(cards, 'Should show at least 1 product').not.toHaveCount(0);
  await expect(cards.first().getByTestId('category-label')).toHaveText('Electronics');
});
\`\`\`

Now write a test for: [your scenario]
```

## 10. Summary & Cheat Sheet

### AI Testing Toolbox

```
Code generation
  npx playwright codegen <url>          → recorded test skeleton
  Claude prompt (user story)            → full spec file
  Claude prompt (HTML snippet)          → targeted tests
  Claude prompt (screenshot)            → visual-to-test

Self-healing
  SmartLocator (multi-strategy)         → fallback at runtime
  SelfHealingPage (AI repair)           → ask Claude for new selector
  Post-CI healing script                → batch repair from report

Autonomous agents
  Simple agent loop (screenshot + LLM)  → goal-driven navigation
  Computer Use API                      → pixel-level control

Maintenance
  migrateLocators.ts                    → batch modernize selectors
  reviewTests.ts                        → quality score + issues
  generateTestData.ts                   → realistic, varied inputs
```

### Prompt Quality Checklist

```
✅ Specify the role: "You are a Playwright engineer..."
✅ State hard constraints: "Never use waitForTimeout"
✅ Provide context: page object, HTML, user story
✅ Specify output format: "Return ONLY valid JSON / TypeScript"
✅ Add examples (few-shot): show 1-2 good examples
✅ Request chain-of-thought for complex logic
✅ Limit scope: "One test per acceptance criterion"
```

### When to Use What

```
Daily coding          → Claude/Copilot in IDE (Layer 1)
New feature coverage  → Prompt-based generation (Layer 2)
Broken selectors      → Self-healing wrapper (Layer 3)
Exploratory testing   → Agent loop / Computer Use (Layer 4)
Regression suite      → Combine Layer 1 + Layer 3
```
